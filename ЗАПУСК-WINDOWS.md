# Запуск сайта НЕОН-ЭК на Windows (для проверки локально)

Цель: открыть живой сайт у себя на компьютере по адресу http://localhost:3000 —
с работающим каталогом, фильтрами, поиском, админкой и заказами.

Делается один раз. Дальше любые правки кода будут видны сразу после обновления страницы.

---

## Шаг 1. Установить Node.js

1. Открой https://nodejs.org
2. Скачай версию **LTS** (большая зелёная кнопка), запусти установщик, жми «Далее» везде.
3. Проверь установку: открой «Командную строку» (нажми Win, набери `cmd`, Enter) и введи:
   ```
   node -v
   ```
   Должна показаться версия (например `v20.x.x`). Если показалась — Node установлен.

---

## Шаг 2. Установить PostgreSQL (база для аккаунтов и заказов)

> Если тебе пока нужен только просмотр **каталога** (без входа в аккаунт и без
> оформления заказов), этот шаг можно пропустить — каталог, фильтры и поиск
> работают без базы. Для заказов и личного кабинета база нужна.

1. Скачай установщик: https://www.postgresql.org/download/windows/ (кнопка
   «Download the installer», версия 16).
2. При установке задай пароль для пользователя `postgres` — **запомни его**.
3. После установки открой приложение **pgAdmin** (ставится вместе с PostgreSQL),
   подключись и создай базу с именем `neon_ek` (правый клик на Databases → Create → Database).

Строка подключения будет такой (подставь свой пароль):
```
postgres://postgres:ТВОЙ_ПАРОЛЬ@localhost:5432/neon_ek
```

---

## Шаг 3. Распаковать проект

1. Распакуй полученный архив `neon-ek-vps.tar.gz` в удобную папку, например `C:\neon`.
   (Windows умеет распаковывать .tar.gz через 7-Zip; если нет — поставь 7-Zip с https://7-zip.org)
2. Должна получиться папка с файлами проекта (внутри есть `package.json`, папки `app`, `lib`, `data`).

---

## Шаг 4. Настроить переменные (файл .env)

1. В папке проекта найди файл `.env.example`, скопируй его и переименуй копию в `.env`
   (просто `.env`, без `.example`).
2. Открой `.env` Блокнотом и заполни:
   ```
   APP_URL=http://localhost:3000
   BETTER_AUTH_SECRET=любая_длинная_случайная_строка_сюда
   DATABASE_URL=postgres://postgres:ТВОЙ_ПАРОЛЬ@localhost:5432/neon_ek
   BITRIX24_WEBHOOK_URL=
   PORT=3000
   ```
   - `BITRIX24_WEBHOOK_URL` оставь пустым пока — заполнишь, когда будешь подключать Битрикс24
     (как получить вебхук — описано в DEPLOY.md, раздел 6).
   - `BETTER_AUTH_SECRET` — просто набери произвольные буквы/цифры, штук 30.

---

## Шаг 5. Установить зависимости и запустить

Открой командную строку **в папке проекта**. Самый простой способ: открой папку
проекта в Проводнике, в адресной строке Проводника набери `cmd` и нажми Enter —
откроется командная строка уже в этой папке.

Введи по очереди:

```
npm install
```
(ставит библиотеки, идёт пару минут — нужен интернет)

Если ставил базу (Шаг 2) — создай таблицы:
```
npm run db:push
```

Запусти сайт:
```
npm run dev
```

Когда увидишь строку вроде `Ready on http://localhost:3000` — открой браузер и
зайди на **http://localhost:3000**. Это твой живой сайт.

Чтобы остановить сайт — в командной строке нажми `Ctrl + C`.
Чтобы запустить снова — снова `npm run dev` в той же папке.

## Что делать дальше

- Любую правку кода, которую я пришлю, ты применяешь в файлах проекта (или я даю
  новый архив), затем обновляешь страницу в браузере — изменения видны сразу.
- Когда сайт будет готов — разворачиваем на хостинг по инструкции в **DEPLOY.md**.

## Если что-то не работает

- Скопируй текст ошибки из командной строки и пришли мне — я подскажу, что исправить.
- `'npm' не является внутренней или внешней командой` → Node не установлен или нужно
  перезапустить командную строку после установки Node.
- Каталог открывается, но висит «Загрузка…» → проверь, что папка `data` с файлом
  `products.json` есть в проекте.
- Не входит в аккаунт / ошибка БД → проверь `DATABASE_URL` в `.env` и что PostgreSQL запущен.
